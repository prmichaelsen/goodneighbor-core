# Task 36: Type Generation Pipeline

**Milestone**: [M9 - OpenAPI Spec, Type Generation & Health](../../milestones/milestone-9-openapi-type-generation.md)
**Estimated Time**: 3 hours
**Dependencies**: Task 35 (OpenAPI spec)
**Status**: Not Started

---

## Objective

Set up `openapi-typescript` to generate `types.generated.ts` from the OpenAPI spec, add convenience type aliases, and create a CI validation script.

---

## Steps

### 1. Install `openapi-typescript`

```bash
npm install -D openapi-typescript
```

### 2. Add `generate:types` script

```json
{
  "scripts": {
    "generate:types": "openapi-typescript docs/openapi.yaml -o src/clients/svc/v1/types.generated.ts"
  }
}
```

### 3. Run generation

```bash
npm run generate:types
```

Verify output at `src/clients/svc/v1/types.generated.ts`.

### 4. Create convenience aliases

`src/clients/svc/v1/types.ts`:
```typescript
import type { components } from './types.generated';

export type CreatePostDto = components['schemas']['CreatePostDto'];
export type PostViewModel = components['schemas']['PostViewModel'];
// ... all schema types as clean exports
```

### 5. Commit generated file

Add `types.generated.ts` to source control with banner comment:
```
// AUTO-GENERATED by openapi-typescript — DO NOT EDIT
// Regenerate: npm run generate:types
```

### 6. CI validation script

Add script or `package.json` command to verify types are up-to-date:
```bash
npm run generate:types && git diff --exit-code src/clients/svc/v1/types.generated.ts
```
